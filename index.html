Desktop Application

Autoshorts Agent Factory runs on the user's desktop, not as a web app.

This website is a public information and support page for the desktop product. Users create videos locally, start account connection from the desktop workflow, and use the official platform authorization flow when connecting a TikTok account.

How Login Works Privacy Policy

What the desktop app does

  • Generates short-form video ideas, scripts, captions, and thumbnails
  • Builds local video exports and publishing metadata on the user's machine
  • Lets the user connect the user's own TikTok account from the desktop workflow
  • Sends content for draft or publishing review only after explicit user action

Website purpose

This website is not the main application interface. It exists to provide public product information, legal pages, and support contact details for platform review and end users.

The product itself runs as a desktop and command-line workflow. Users do not create projects, render videos, or complete their work from this website.

Why there is no web login form

Login is initiated from the desktop application because the product is not a browser-based SaaS. The website is intentionally informational and therefore does not present a standalone username and password experience.

When a user wants to connect TikTok, the desktop app starts the official authorization flow and the user signs in with TikTok directly through the provider's own authentication experience.

How TikTok login works

  1. The user launches the desktop application on the user's own computer.
  2. The user chooses the TikTok connection command inside the desktop or CLI workflow.
  3. The app opens the official TikTok authorization page so the user can sign in and approve access.
  4. After approval, the desktop workflow stores the returned authorization data locally and uses it only for the user's requested posting actions.

The website does not authenticate users directly because authentication belongs to the desktop product and the official provider flow.

Available user actions

  • Create and prepare short vertical videos locally
  • Review scripts, captions, thumbnails, and metadata before posting
  • Connect a TikTok account owned by the user through official authorization
  • Send content for draft creation or final publishing review after user confirmation

Intended users

The desktop workflow is designed for creators, editors, and channel operators who want to streamline short-video production while keeping human review in the loop.

Users remain responsible for the content they create, the media rights they use, and the final decision to publish on connected platforms.
